# DOCX Production Principles

These principles prepare Markdown so it can later become a clean Word/DOCX document.

## Structure

- Use real heading hierarchy. Do not skip from H1 to H4 for visual styling.
- Use real lists, not typed bullet characters.
- Use section-level page breaks only between logical sections.
- Keep one idea per paragraph.

## Tables

- Use tables for real comparisons, not visual layout.
- Add a short caption that explains how to read the table.
- Keep column headers as actual comparison axes.
- Avoid very wide tables; split them or mark them as landscape candidates.
- Keep cells concise. Long prose belongs in paragraphs or bullets.

## Images

- Add meaningful alt text.
- Add a caption when the image is evidence.
- Keep screenshots close to their explanation.
- Avoid making essential meaning depend on image-only text.

## Long Documents

- Use a table of contents when headings exceed three major sections.
- Put source notes near claims.
- Preserve code, links, numbers, and identifiers exactly.
- Avoid interactive-only constructs such as collapsible details for essential content.

## Handoff Checklist

- Headings can produce a useful outline.
- Lists are semantic.
- Tables have captions.
- Images have alt/caption context.
- Page breaks are logical and not inside content blocks.
- No layout-only tables.
