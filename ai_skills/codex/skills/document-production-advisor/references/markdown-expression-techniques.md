# Markdown Expression Techniques

These techniques are designed for Markdown Pattern Studio's existing class attribute syntax.

## Highest Priority

1. `.safe-zone`
   - Use when the document will become a slide, hero, banner-like page, or blog embed.
   - Keeps essential content in a central reading area.

2. `.feature-grid`
   - Use for 3-6 peer items.
   - Better than a plain list when the items are independent capabilities.

3. `.big-number-hero`
   - Use for one memorable metric, not a table of metrics.
   - Add one supporting sentence under the number.

## Other Useful Techniques

- `.problem-statement`: problem/pain framing.
- `.metrics-dashboard`: KPI bullets.
- `.contrast-pair`: before/after or old/new comparison.
- `.gradient-number`: accent numeric emphasis.
- `.oversized`: one-line typographic statement.
- `.screenshot-shadow`: product/result screenshot emphasis.

## Rules

- Do not apply expressive classes to every section.
- Do not use `.big-number-hero` for more than one page in a row.
- Use `.feature-grid` only when list items are comparable in weight.
- Use `.contrast-pair` only for true pairs.
- Keep blog embeds stack-friendly; grids must collapse cleanly on narrow screens.
