# Expression Utilities {#cover .cover .dark eyebrow="Visual QA"}

Renderer-supported document techniques for request-checked outputs.

---
{: .page-break}

## Cost of Drift {: .problem-statement .safe-zone}

If the document looks polished but misses the user's actual request, it fails. The output needs a visible contract, a traceable section plan, and a render check.

---
{: .page-break}

## 42% {: .big-number-hero}

Fewer rewrite cycles after checking the rendered artifact instead of only reading source Markdown.

---
{: .page-break}

## Capability Map {: .feature-grid}

- Request contract
- Section trace
- Rendered UX check
- Blog-safe export
- DOCX handoff structure
- Unsupported idea removal

---
{: .page-break}

## Quality Signals {: .metrics-dashboard}

- Format | HTML / Blog / DOCX | Target known
- Layout | Desktop + narrow | Checked visually
- Evidence | Tables / images / code | Preserved
- Result | No clipping | Pass only after render

---
{: .page-break}

## Before And After {: .contrast-pair}

Before: the document is visually decorated, but nobody can trace whether it answered the request.

After: every section maps to a requirement, and the rendered page is checked for visible breakage.
